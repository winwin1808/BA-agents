# Socratic Questioning Framework for PRDs

Use this reference during PRD discovery, refinement, or review when feature rationale is still fuzzy. The goal is to improve clarity, not to interrogate stakeholders.

## Purpose

- Uncover assumptions
- Clarify fuzzy thinking
- Surface risks early
- Strengthen rationale

## How to Use It

- Ask 3-5 questions per feature
- Start broad with the problem, then narrow into solution and success, then close on constraints and strategy
- If answers are weak, ask for evidence, examples, or concrete scenarios
- If the PM is stuck, offer options, examples, or hypotheses to react to
- Stop once the PRD has enough clarity to support a decision

## Question Categories

### 1. Problem Clarity

- What specific user pain point does this solve?
- How do we know this is a real problem?
- Who feels this problem most acutely?
- What is the cost of not solving it?

### 2. Solution Validation

- Why is this the right solution?
- What alternatives were considered and rejected?
- What is the simplest version that works?
- How will users discover this feature?

### 3. Success Criteria

- How will we know this is successful?
- What would make this a failure?
- Which metric should move, and by how much?
- What adoption target is realistic?

### 4. Constraints and Trade-offs

- What are technical constraints and risks?
- What is explicitly out of scope?
- What existing workflows are affected?
- If resources are halved, what gets cut?

### 5. Strategic Fit

- Why build this now?
- How does this support broader strategy?
- What if we wait 6 months?
- How does this affect competitive position?

## Recommended Flow

1. Start with 1-2 problem questions
2. Ask 1 solution question
3. Ask 1 success question
4. Ask 1 constraint or strategic-fit question
5. Add follow-up questions only where the answer is weak or unsupported

## Output Goal

After questioning, the PRD should have:

- A clear problem statement with evidence
- Rationale for the chosen solution
- Measurable success criteria
- Explicit scope boundaries
- A strategic narrative for timing

## Facilitation Notes for AI

- Keep the tone collaborative and concise
- Prefer short, high-signal questions over long questionnaires
- Do not ask every question in this file
- Choose the smallest set of questions that exposes the main uncertainty
- If the stakeholder already gave strong evidence, summarize it and move forward instead of re-asking
